# Codex 提示词模板（Chrome 探测 + SessionHub 落地）

## 使用目的
当我要新增一个接口自动化任务时，我希望你先用 `@chrome` 快速探测接口，再把可稳定运行的方案落到 `sessionhub(9222专用浏览器)`，最终交付可直接执行的自动化脚本。

---

## 一键提示词（可直接复制）

你现在按下面流程执行，不要只给方案，要直接落代码并验证：

1. 使用 `@chrome` 在我已登录的主力 Chrome 中完成前期探测：
- 进入目标页面
- 触发查询/翻页/导出等动作
-识别真实请求的 `url`、`method`、关键 `headers`、`body` 字段、分页参数、成功/失败特征
- 总结接口触发条件（用户要点哪里才会发请求）

2. 将探测结果落地到我的 `sessionhub`（`9222` 专用浏览器）：
- 新增或更新对应 scene 配置
- 保持 `check -> capture -> recheck -> run` 的流程
- 不要依赖我手工复制 curl
- 明确失效时的报错文案和重抓提示

3. 代码实现要求：
- 兼容我现有调用方式，不随意改下游入口
- 只做必要改动，避免影响已有业务逻辑
- 输出文件路径、日志路径、失败定位方法必须写清楚
- 涉及猫超/聚水潭场景时，优先保证稳定性和可复用性

4. 验收要求（必须执行）：
- 在 `sessionhub(9222)` 环境完成一次真实可用验证
- 给出“成功/失败”的明确结论
- 若失败，给出可执行修复步骤，不要停留在抽象描述

5. 最终交付格式：
- 改动摘要（改了什么、为什么）
- 运行命令（可直接复制执行）
- 输出落点（文件保存到哪里）
- 风险与注意事项（只写和本任务相关的）

---

## 我的固定偏好（你必须遵守）
- 全程中文，简洁直接
- 优先给可执行命令，不要长篇理论
- 前期可用 `@chrome` 提速，但最终必须以 `9222 sessionhub` 的可运行结果为准
- 不要让我手工抓包、复制 curl，除非你明确说明当前步骤为什么无法绕过

---

## 我发需求时可补充的变量（可选）
- 目标系统：`猫超 / 聚水潭 / 其他`
- 目标页面：`URL 或页面名称`
- 目标动作：`查询/导出/翻页/筛选`
- 期望产物：`Excel/CSV/JSON/写回表格`
- 是否夜间自动执行：`是/否`
- 是否要失败通知：`微信/日志/两者`
